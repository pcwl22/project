# 忘记密码功能说明

## 功能概述

忘记密码功能允许用户通过回答安全问题来重置密码，无需管理员介入。

## 核心特性

✅ **三步验证流程**：用户名验证 → 安全问题回答 → 密码重置  
✅ **安全问题保护**：基于用户设置的安全问题验证身份  
✅ **友好的向导界面**：清晰的步骤指示和表单验证  
✅ **即时生效**：密码重置后立即可用新密码登录  

## 使用指南

### 用户注册时设置安全问题

1. 在注册页面填写基本信息
2. 选择一个安全问题（可选）
3. 输入安全答案
4. 完成注册

**可选的安全问题**：
- 您的出生地是？
- 您母亲的姓名是？
- 您的小学名称是？
- 您最喜欢的颜色是？
- 您的宠物名字是？

### 忘记密码流程

#### 步骤1：验证用户名
1. 在登录页面点击"忘记密码？"链接
2. 输入您的用户名
3. 点击"下一步"
4. 系统显示您设置的安全问题

#### 步骤2：回答安全问题
1. 查看显示的安全问题
2. 输入您的安全答案
3. 点击"下一步"

#### 步骤3：设置新密码
1. 输入新密码（至少6位）
2. 再次输入新密码确认
3. 点击"完成"
4. 密码重置成功，返回登录页面

### 使用新密码登录
1. 在登录页面输入用户名
2. 输入新密码
3. 点击"登录"

## 技术实现

### 数据库字段

在 `users` 表中添加了以下字段：

```sql
email VARCHAR(255)              -- 邮箱（预留）
security_question VARCHAR(255)  -- 安全问题
security_answer VARCHAR(255)    -- 安全答案
```

### API 接口

#### 1. 忘记密码 - 第一步
```
POST /api/auth/forgot-password
```

**请求体**：
```json
{
  "username": "user123"
}
```

**响应**：
```json
{
  "success": true,
  "message": "请回答安全问题",
  "security_question": "您的出生地是？"
}
```

#### 2. 重置密码 - 第二步
```
POST /api/auth/reset-password
```

**请求体**：
```json
{
  "username": "user123",
  "security_answer": "北京",
  "new_password": "newpass123",
  "confirm_password": "newpass123"
}
```

**响应**：
```json
{
  "success": true,
  "message": "密码重置成功，请使用新密码登录"
}
```

## 安全机制

### 1. 答案验证
- 不区分大小写
- 必须完全匹配
- 错误时给出提示

### 2. 密码要求
- 最少6位字符
- 必须确认密码
- 两次输入必须一致

### 3. 用户验证
- 验证用户名存在
- 检查是否设置安全问题
- 未设置则提示联系管理员

## 常见问题

### Q1: 注册时没有设置安全问题怎么办？
**A**: 如果注册时未设置安全问题，忘记密码时系统会提示"该用户未设置安全问题，请联系管理员重置密码"。建议所有用户注册时设置安全问题。

### Q2: 忘记了安全答案怎么办？
**A**: 如果忘记安全答案，需要联系系统管理员手动重置密码。

### Q3: 安全答案区分大小写吗？
**A**: 不区分。系统会将答案转换为小写进行比较。

### Q4: 可以修改安全问题吗？
**A**: 当前版本不支持修改安全问题。如需修改，请联系管理员。

### Q5: 密码重置后多久生效？
**A**: 立即生效。重置成功后可以马上使用新密码登录。

## 最佳实践

### 选择安全问题
1. 选择只有您知道答案的问题
2. 避免选择容易被猜到的问题
3. 答案不要过于简单

### 设置安全答案
1. 使用容易记住但难以猜测的答案
2. 避免使用常见答案
3. 记住您的答案（不区分大小写）

### 密码安全
1. 使用强密码（字母+数字+符号）
2. 定期更换密码
3. 不要与其他网站使用相同密码

## 管理员操作

### 手动重置用户密码

如果用户未设置安全问题或忘记安全答案，管理员可以直接在数据库中重置：

```sql
-- 重置密码为 123456
UPDATE users 
SET password = '123456' 
WHERE username = 'user123';
```

### 为用户设置安全问题

```sql
UPDATE users 
SET security_question = '您的出生地是？',
    security_answer = '北京'
WHERE username = 'user123';
```

### 查看用户安全问题设置

```sql
SELECT username, security_question, 
       CASE WHEN security_answer IS NOT NULL THEN '已设置' ELSE '未设置' END as answer_status
FROM users;
```

## 故障排查

### 问题：点击"忘记密码"没有反应
**解决方案**：
1. 检查浏览器控制台错误
2. 确认后端服务正在运行
3. 检查网络连接

### 问题：提示"用户未设置安全问题"
**解决方案**：
1. 联系管理员重置密码
2. 或在数据库中手动添加安全问题

### 问题：安全答案正确但提示错误
**解决方案**：
1. 检查答案是否有多余空格
2. 确认答案与注册时一致
3. 查看数据库中的实际答案
