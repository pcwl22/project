# 空货架检测算法说明

## 算法概述

本系统采用"单模型检测 + 智能推算"的方案，只需要一个商品检测模型，通过几何分析算法自动推算空货架位置。

## 核心优势

✅ **无需训练空货架样本**：不需要收集和标注空货架数据  
✅ **适应性强**：自动适应不同货架布局和商品尺寸  
✅ **准确率高**：基于几何关系，误报率低  
✅ **实时处理**：算法简单高效，处理速度快  
✅ **可配置**：支持参数调整，适应不同场景  

## 算法流程

### 1. 商品检测阶段
使用 YOLOv8 模型检测图片中的所有商品：
```
输入图片 → YOLO模型 → 商品检测框 + 置信度
```

### 2. 行聚类阶段
将检测到的商品按垂直位置分组到不同货架层：

```python
# 按 Y 坐标中心点排序
items_sorted = sorted(items, key=lambda x: (x[2] + x[4]) / 2)

# 聚类成行
for item in items_sorted:
    if abs(cur_y_center - last_y_center) < avg_height * row_threshold:
        # 同一行
        current_row.append(item)
    else:
        # 新的一行
        rows.append(current_row)
        current_row = [item]
```

**聚类条件**：
- 如果两个商品的 Y 坐标中心点距离 < 平均高度 × `row_threshold`
- 则认为它们在同一层货架

### 3. 间隙检测阶段
在每一行中检测商品之间的水平间隙：

```python
# 按 X 坐标排序
row_sorted = sorted(row, key=lambda x: x[1])

# 检测间隙
for i in range(len(row_sorted) - 1):
    gap_width = row_sorted[i+1][x1] - row_sorted[i][x2]
    
    if gap_width > avg_width * gap_threshold:
        # 发现空货架
        create_empty_shelf(gap_start, gap_end, row_y1, row_y2)
```

**判定条件**：
- 间隙宽度 > 平均商品宽度 × `gap_threshold`
- 且间隙宽度 > `min_gap_pixels`（最小像素阈值）

### 4. 边缘检测阶段（可选）
检测货架左右边缘的空位：

```python
# 左边缘
if first_product_x1 > avg_width * gap_threshold:
    create_empty_shelf(0, first_product_x1, row_y1, row_y2)

# 右边缘
if img_width - last_product_x2 > avg_width * gap_threshold:
    create_empty_shelf(last_product_x2, img_width, row_y1, row_y2)
```

### 5. 结果可视化
绘制检测结果并添加统计信息：
- 商品框：绿色，线宽 2px
- 空货架框：红色，线宽 3px
- 统计信息：黄色文字，左上角显示

## 算法参数

### 1. gap_threshold（间隙阈值）
- **默认值**：0.5
- **含义**：商品宽度的倍数
- **作用**：控制空位检测的灵敏度
- **调整建议**：
  - 增大（如 0.7）：更保守，减少误报
  - 减小（如 0.3）：更灵敏，增加检出率

### 2. row_threshold（行聚类阈值）
- **默认值**：0.6
- **含义**：商品高度的倍数
- **作用**：控制行聚类的容错性
- **调整建议**：
  - 增大（如 0.8）：允许更大的垂直偏差
  - 减小（如 0.4）：更严格的行划分

### 3. min_gap_pixels（最小间隙像素）
- **默认值**：20
- **含义**：最小间隙像素数
- **作用**：过滤过小的间隙
- **调整建议**：
  - 增大（如 30）：忽略更小的间隙
  - 减小（如 10）：检测更小的空位

### 4. edge_detection（边缘检测）
- **默认值**：True
- **含义**：是否检测货架边缘空位
- **作用**：检测货架左右两端的空位
- **调整建议**：
  - True：检测边缘空位（推荐）
  - False：只检测商品之间的空位

### 5. min_products_per_row（最少商品数）
- **默认值**：2
- **含义**：每行最少商品数
- **作用**：过滤商品过少的行
- **调整建议**：
  - 增大（如 3）：只处理商品较多的行
  - 减小（如 1）：处理所有行

## 算法示例

### 场景一：标准货架
```
商品分布：[商品1] [商品2] [空位] [商品3] [商品4]
平均宽度：50px
间隙宽度：80px

判定：80px > 50px × 0.5 = 25px
结果：检测到空货架 ✅
```

### 场景二：密集货架
```
商品分布：[商品1][商品2][商品3][商品4]
平均宽度：50px
间隙宽度：5px

判定：5px < 50px × 0.5 = 25px
结果：无空货架 ✅
```

### 场景三：多层货架
```
第1层：[商品1] [空位] [商品2]
第2层：[商品3] [商品4] [商品5]
第3层：[空位] [商品6] [空位]

结果：
- 第1层：检测到 1 个空位
- 第2层：无空位
- 第3层：检测到 2 个空位（含边缘）
```

## 常见问题

### Q1: 为什么有些空位没检测到？
A: 可能是间隙太小，尝试降低 `gap_threshold` 或 `min_gap_pixels`

### Q2: 为什么有误报？
A: 可能是阈值太低，尝试提高 `gap_threshold` 或 `min_gap_pixels`

### Q3: 多层货架识别不准？
A: 调整 `row_threshold`，增加垂直容错性

### Q4: 边缘空位不需要检测？
A: 设置 `edge_detection = False`

### Q5: 如何提高检测速度？
A: 降低图片分辨率，或使用更小的 YOLO 模型（如 YOLOv8n）
