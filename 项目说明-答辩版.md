# 项目说明（答辩版）

## 1. 项目名称

基于 YOLOv8 的货架商品检测与空货架识别系统

## 2. 项目一句话介绍

这是一个面向超市和便利店货架巡检场景的智能检测系统，能够对图片、视频和摄像头画面中的商品与空货架情况进行自动识别，并提供历史记录、模型管理和参数配置功能。

## 3. 项目要解决的问题

- 人工巡检效率低
- 缺货发现不及时
- 巡检结果难以量化保存
- 传统模型演示缺少业务系统支撑

## 4. 我的解决方案

我采用了“YOLOv8 目标检测 + 空货架推断算法 + 前后端分离系统”的方案：

- 用 YOLOv8 识别商品和相关目标
- 用自定义空货架推断逻辑补足货架空位分析
- 用 FastAPI 提供后端接口
- 用 Vue 3 实现可操作的前端系统
- 用 MySQL 保存用户与检测历史

## 5. 系统主要功能

### 图片检测

- 上传单张图片
- 返回检测框结果
- 统计商品数、空货架数和空置率

### 视频检测

- 上传视频
- 输出处理后视频
- 支持结果预览和下载

### 摄像头实时检测

- 实时检测货架状态
- 动态显示空货架数量
- 会话结束后保存最具代表性结果

### 历史记录

- 保存检测记录
- 查看详情
- 删除记录

### 模型管理

- 查看可用模型
- 上传新的 `.pt` 模型
- 切换当前使用模型

### 算法参数配置

- 在线调整阈值
- 调整空货架检测参数
- 支持配置持久化保存

### 用户管理

- 登录注册
- 找回密码
- 管理员查看和删除用户

## 7. 系统结构

系统采用前后端分离架构：

1. 前端负责页面展示和交互
2. 后端负责接口、推理和数据处理
3. 数据库存储用户与检测记录
4. 本地文件保存图片、视频和结果文件

## 8. 项目亮点

### 亮点 1：不仅做目标检测，还做业务系统

这个项目不是单纯训练一个模型，而是把模型真正做成了一个可以登录、上传、检测、查看历史和调参数的完整系统。

### 亮点 2：支持三种输入场景

- 图片检测
- 视频检测
- 摄像头实时检测

相比只做单图推理，系统展示效果更完整。

### 亮点 3：实现了空货架推断算法

除了使用 YOLOv8 检测商品，我还在后端加入了空货架推断逻辑，主要通过商品之间的间隙、行聚类和边缘区域分析来判断空位。

### 亮点 4：支持参数可调和结果可追溯

- 参数可以在线配置并保存
- 检测历史可以存入数据库
- 结果可以回查和删除

这让系统更接近真实应用，而不仅是实验代码。

### 亮点 5：视频处理做了性能优化

项目对视频检测加入了：

- GPU 支持
- 跳帧处理
- 分辨率缩放
- 快速绘制检测框
- 可选 FFmpeg 转码

## 9. 我在项目中完成的核心工作

- 设计并实现前后端分离架构
- 接入 YOLOv8 模型并封装检测服务
- 编写空货架推断算法
- 实现图片、视频、摄像头三种检测流程
- 设计检测历史存储结构
- 实现模型管理与参数配置模块
- 完成用户登录、找回密码和管理员管理功能

## 10. 当前不足

- 管理员权限判断方式还比较简单
- 密码加密方式不够完善
- 部分路径仍依赖本机环境
- 前端接口地址配置还可以继续统一

## 11. 后续改进方向

- 增加更规范的权限系统
- 增强密码安全性
- 支持 Docker 部署
- 增加数据图表分析
- 提升跨机器部署的便利性

## 12. 总结

本项目完成了从目标检测模型到业务系统落地的完整实现，既体现了算法能力，也体现了系统设计与工程实现能力。整体上，它不仅可以作为一个检测演示项目，也具备进一步扩展为实际零售巡检系统的基础。
