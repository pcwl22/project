# 项目说明（论文版）

## 1. 项目名称

基于 YOLOv8 的货架商品检测与空货架识别系统设计与实现

## 2. 项目研究背景

在超市、便利店、仓储货架等零售场景中，货架巡检、缺货识别和商品陈列检查通常依赖人工完成。这种方式虽然实现简单，但在实际应用中存在明显不足。首先，人工巡检需要投入较多的人力成本，特别是在货架数量较多、巡检频率较高的场景下，效率较低。其次，不同巡检人员在判断标准上存在差异，容易导致巡检结果不一致，缺乏统一性和客观性。再次，人工方式往往具有滞后性，无法及时发现缺货、空位或陈列异常问题，从而影响补货效率和门店运营质量。

随着计算机视觉与深度学习技术的快速发展，目标检测技术在零售场景中的应用逐渐成熟。YOLO 系列模型因检测速度快、精度较高、部署方便，被广泛应用于商品识别、货架分析和视频监控等任务中。因此，设计并实现一个基于 YOLOv8 的货架商品检测与空货架识别系统，具有较强的现实意义和应用价值。

## 3. 项目研究目的

本项目旨在构建一套面向货架巡检场景的智能检测系统，通过引入 YOLOv8 目标检测模型，对货架上的商品进行识别，并结合空货架推断算法，实现对货架空位情况的自动分析。项目不仅关注检测模型本身，还将模型能力与业务系统相结合，完成从检测、展示、记录、管理到配置调优的完整系统实现。

具体目标包括：

- 实现图片、视频和摄像头三种输入方式下的货架检测功能
- 实现商品检测结果的可视化展示
- 实现空货架数量、总货位数量和空置率的统计分析
- 实现检测历史记录保存与查询
- 实现模型管理与算法参数配置功能
- 构建完整的前后端分离系统，提升项目的实用性和展示性

## 4. 系统总体设计

本项目采用前后端分离架构设计。前端负责界面展示、用户交互和文件上传；后端负责业务逻辑处理、YOLO 模型推理、数据库读写以及结果返回；数据库负责保存用户和检测记录信息；本地文件系统用于存储原始文件和检测结果文件。

系统整体工作流程如下：

1. 用户通过前端页面完成登录或注册。
2. 用户在图片检测、视频检测或摄像头检测页面中提交待检测数据。
3. 前端将请求发送至 FastAPI 后端接口。
4. 后端调用 YoloService 完成目标检测与空货架推断。
5. 检测结果保存到本地文件系统与数据库中。
6. 后端将检测结果、统计信息和访问路径返回给前端。
7. 前端完成结果可视化展示，并支持历史记录查看、模型切换和参数配置。

## 5. 系统功能设计

### 5.1 用户认证功能

系统支持用户注册、登录、忘记密码和密码重置功能。用户注册时需要填写用户名、密码以及安全问题和安全答案；登录成功后系统根据用户名中是否包含 `admin` 来判断用户是否具有管理员权限；忘记密码时通过安全问题验证后可重置密码。

### 5.2 图片检测功能

图片检测模块支持用户上传单张图片，选择检测模型，调整置信度阈值和 IOU 阈值，并执行目标检测。检测完成后，系统会返回带检测框的结果图，同时统计商品数量、空货架数量和空置率。前端还支持将统计结果导出为 CSV 文件。

### 5.3 视频检测功能

视频检测模块支持用户上传本地视频文件，后端对视频逐帧或跳帧进行检测，并生成处理后的视频结果。用户可在前端直接预览检测视频，并将结果下载到本地。

### 5.4 摄像头实时检测功能

摄像头检测模块支持获取本地摄像头列表并进行实时检测。系统会在视频流上叠加检测框和统计信息，实时显示商品数量、空货架数量、最大空货架数和空置率。检测结束后，系统会保存空货架数量最多的一帧作为本次会话的代表结果。

### 5.5 历史记录管理功能

系统支持保存和展示检测历史记录。用户可以查看图片、视频和摄像头检测的历史数据，查看单条记录的详细信息，并删除指定历史记录及关联本地文件。

### 5.6 模型管理功能

系统支持获取当前可用模型列表，并支持管理员上传新的 `.pt` 模型权重文件。上传完成后，前端模型列表可自动刷新，用户可切换当前使用的检测模型。

### 5.7 算法配置功能

系统支持在线调整空货架检测相关参数，包括空货架检测是否启用、间隙阈值、行聚类阈值、最小空隙像素、边缘检测开关等。修改后的参数将被持久化保存至 `algorithm_config.json` 文件中。

### 5.8 用户管理功能

管理员可查看用户列表、用户是否为管理员以及删除普通用户。该模块为系统的后台管理提供了基本支撑。

## 6. 技术方案

### 6.1 前端技术

前端采用 Vue 3 作为核心框架，使用 TypeScript 提升类型安全，使用 Vite 作为构建工具，使用 Element Plus 实现界面组件，并通过 Axios 与后端进行接口通信。

### 6.2 后端技术

后端采用 FastAPI 构建 RESTful API，使用 SQLAlchemy 作为 ORM 框架，使用 Pydantic 进行数据结构定义和参数校验，使用 PyMySQL 与 MySQL 数据库进行连接，使用 Uvicorn 运行后端服务。

### 6.3 检测与图像处理技术

项目核心检测能力基于 Ultralytics 提供的 YOLOv8 模型实现，并结合 PyTorch、OpenCV、NumPy 与 Pillow 实现图像处理、视频处理和结果可视化。

## 7. 数据库设计

根据项目代码，系统主要包含以下三类核心数据表：

### 7.1 用户表 `users`

用于保存用户基础信息，包括用户名、密码、邮箱、安全问题、安全答案及创建时间。

### 7.2 检测结果表 `detection_results`

用于保存单次检测的主记录，包括用户 ID、原始文件路径、结果文件路径、文件类型、最大空货架数及创建时间。

### 7.3 检测框表 `detection_boxes`

用于保存单次检测中的每一个检测框信息，包括类别名称、坐标、置信度等。

## 8. 空货架识别算法设计

本项目的关键创新点之一在于：除了直接使用 YOLOv8 对商品进行目标检测外，还实现了基于货架陈列规律的空货架推断算法。

该算法在 `backend/app/services/yolo_service.py` 中实现，主要流程如下：

1. 使用 YOLOv8 对输入图像进行目标检测，得到商品和可能的空货架检测结果。
2. 对位于黑边区域的异常检测框进行过滤，减少误检。
3. 根据检测到的商品位置，按垂直方向聚类形成多行货架结构。
4. 在每一行中，根据相邻商品之间的横向间隙大小，判断是否存在空货架区域。
5. 对货架左右两侧的边缘区域进行空位检测。
6. 去除与商品检测框明显重叠的空货架框，减少冲突。
7. 输出最终的商品框与空货架框结果，并统计空货架数量和空置率。

该算法支持通过参数调节适配不同货架场景，具有较强的灵活性。

## 9. 视频检测优化设计

为了提升视频检测效率，项目在实现中加入了多项性能优化策略，包括：

- 支持 GPU 推理
- 支持跳帧检测
- 支持处理分辨率缩放
- 支持快速绘制检测框
- 支持可选的 FFmpeg 二次转码

这些优化手段有助于提高系统在视频场景下的运行效率和可用性。

## 10. 前端界面设计

前端主页面由 `App.vue` 统一控制，登录成功后通过左侧导航栏切换不同功能页面，包括图片检测、视频检测、摄像头检测、历史记录、算法设置、模型管理和用户管理等模块。

各页面功能明确，界面交互完整，能够较好地支撑系统演示与用户操作。

## 11. 项目运行与部署说明

项目运行依赖如下环境：

- Python 3.9 及以上
- Node.js 18 及以上
- MySQL 8.0 及以上

后端依赖通过 `requirements.txt` 安装，前端依赖通过 `npm install` 安装。系统支持分别启动前后端，也支持通过 `start.py` 一键启动。

## 12. 当前实现存在的问题

虽然项目已经实现了较完整的业务闭环，但从代码角度看，仍存在一些可改进之处：

- 管理员权限判断方式较简单，目前通过用户名是否包含 `admin` 判断
- 密码校验方式兼容明文与 MD5，安全性不足
- 部分接口的用户过滤逻辑仍较简化
- 项目中仍存在本机绝对路径，换机器部署前需要修改配置
- 前端请求地址存在相对路径和硬编码地址混用问题

## 13. 项目总结

本项目基于 YOLOv8 实现了一个面向货架巡检场景的智能检测系统。项目完成了从模型推理、空货架识别、参数配置、历史记录管理到用户交互界面的完整实现，具有较好的系统完整性和实际展示价值。相较于单纯的目标检测实验，本项目更强调算法能力与业务系统的结合，能够更好地体现毕业设计中“设计与实现”的综合能力。
