# 视频检测功能说明

## 功能概述

视频检测功能允许用户上传监控视频，系统会逐帧分析视频内容，检测货架上的商品并识别空货架区域，最终生成带有标注的检测结果视频。

## 功能特点

### 1. 视频处理
- 支持常见视频格式（MP4、AVI、MOV 等）
- 逐帧检测分析
- 保持原视频帧率和分辨率
- 自动优化处理速度（隔帧处理）

### 2. 实时标注
- 商品检测框（绿色）
- 空货架检测框（红色）
- 置信度显示
- 实时统计信息叠加

### 3. 参数配置
- 置信度阈值调整（0-1）
- IOU 阈值调整（0-1）
- 模型选择
- 与图片检测使用相同的算法参数

## 使用指南

### 1. 上传视频

1. 点击"视频检测"标签
2. 选择要使用的 YOLO 模型
3. 点击"选择视频"按钮
4. 从本地选择视频文件
5. 视频会在左侧预览区域显示

### 2. 调整参数

在开始检测前，可以调整以下参数：

- **置信度阈值**：控制检测结果的最低置信度
  - 默认值：0.25
  - 建议范围：0.2-0.5
  - 值越高，过滤越严格，误检越少

- **IOU 阈值**：控制重叠框的过滤
  - 默认值：0.45
  - 建议范围：0.3-0.6
  - 值越小，过滤越严格

### 3. 开始检测

1. 调整好参数后，点击"开始检测"按钮
2. 系统开始处理视频（可能需要较长时间）
3. 处理完成后，结果视频会在右侧显示
4. 可以播放查看检测结果

### 4. 查看结果

检测完成后：
- 左侧显示原始视频
- 右侧显示标注后的视频
- 可以对比查看检测效果
- 支持播放控制（播放/暂停/进度条）

## 技术实现

### 处理流程

```
1. 上传视频文件
   ↓
2. 后端接收并保存
   ↓
3. 打开视频流
   ↓
4. 逐帧读取
   ↓
5. 每帧执行检测
   - 商品检测（YOLO）
   - 空货架推算（算法）
   ↓
6. 绘制检测结果
   - 检测框
   - 标签
   - 统计信息
   ↓
7. 写入输出视频
   ↓
8. 返回结果视频
```

### 性能优化

1. **隔帧处理**
   - 默认每隔一帧处理一次
   - 大幅提升处理速度
   - 对检测效果影响较小

2. **批量处理**
   - 使用 OpenCV 高效读写
   - GPU 加速（如果可用）
   - 内存优化

3. **进度显示**
   - 在视频上显示当前帧数
   - 用户可以了解处理进度

## API 接口

### 视频检测接口

```http
POST /api/detect
Content-Type: multipart/form-data

Parameters:
- file: 视频文件
- mode: "video"
- conf: 置信度阈值（可选，默认 0.25）
- iou: IOU 阈值（可选，默认 0.45）
- model: 模型名称（可选，默认 "best.pt"）
```

响应：
```
Content-Type: video/mp4
Body: 处理后的视频文件（二进制流）
```

## 注意事项

### 1. 视频大小限制
- 建议视频大小不超过 500MB
- 过大的视频会导致处理时间过长
- 可以先压缩或裁剪视频

### 2. 处理时间
- 处理时间取决于：
  - 视频长度
  - 视频分辨率
  - 帧率
  - 硬件性能
- 1分钟的 1080p 视频大约需要 2-5 分钟处理

### 3. 格式支持
- 推荐格式：MP4（H.264 编码）
- 支持格式：AVI、MOV、MKV 等
- 输出格式：MP4（H.264 编码）

### 4. 浏览器兼容性
- 需要浏览器支持 HTML5 video
- 推荐使用 Chrome、Edge、Firefox 最新版本

## 常见问题

### Q: 视频检测需要多长时间？
A: 取决于视频长度和分辨率。通常 1 分钟的 1080p 视频需要 2-5 分钟处理。

### Q: 为什么处理后的视频文件很大？
A: 系统使用 mp4v 编码器，压缩率较低。可以使用视频压缩工具进一步压缩。

### Q: 可以实时处理视频流吗？
A: 当前版本不支持实时视频流。请使用"摄像头检测"功能进行实时检测。

### Q: 检测结果不准确怎么办？
A: 尝试调整置信度和 IOU 阈值，或使用更适合的模型。

### Q: 视频检测会保存到历史记录吗？
A: 当前版本视频检测不会保存到历史记录，仅图片检测会保存。

### Q: 检测结果视频无法播放怎么办？
A: 可能是编码器兼容性问题。解决方法：
1. 运行 `python test_video_codec.py` 检查可用编码器
2. 确保安装了 ffmpeg
3. 尝试使用其他浏览器（推荐 Chrome）
4. 检查视频文件是否生成成功

## 故障排除

### 视频无法播放

**症状**：检测完成后，结果视频显示黑屏或无法播放

**可能原因**：
1. 视频编码器不兼容
2. 浏览器不支持该编码格式
3. 视频文件损坏

**解决方法**：

1. **检查编码器**
   ```bash
   cd project
   python test_video_codec.py
   ```

2. **安装 ffmpeg**（推荐）
   - Windows: 下载 https://ffmpeg.org/download.html
   - 添加到系统 PATH
   - 重启后端服务

3. **使用其他浏览器**
   - Chrome（推荐）
   - Edge
   - Firefox

4. **检查视频文件**
   - 查看 `project/history/detections/` 目录
   - 确认 `res_*.mp4` 文件存在
   - 尝试用本地播放器打开

### 处理速度慢

**症状**：视频处理时间过长

**解决方法**：
1. 降低视频分辨率
2. 缩短视频长度
3. 使用 GPU 加速（需要 CUDA）
4. 调整隔帧处理参数

### 内存不足

**症状**：处理大视频时程序崩溃

**解决方法**：
1. 分段处理视频
2. 降低视频分辨率
3. 增加系统内存
4. 关闭其他程序
