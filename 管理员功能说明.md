# 管理员功能说明

## 功能概述

管理员用户可以访问系统管理功能，包括用户管理、系统统计等。

## 管理员识别

系统通过用户名判断是否为管理员：
- 用户名包含 "admin"（不区分大小写）的用户自动识别为管理员
- 例如：`admin`、`Admin123`、`administrator` 等

## 功能列表

### 1. 用户管理

管理员可以查看和管理所有用户。

#### 查看用户列表
- 显示所有注册用户
- 显示用户 ID、用户名、管理员标识、注册时间
- 显示检测次数（当前版本暂不支持）

#### 删除用户
- 可以删除普通用户
- 不能删除管理员账号（防止误删）
- 删除用户会级联删除其相关数据

### 2. 系统统计

查看系统整体运行情况：
- 总用户数
- 总检测次数
- 其他统计信息

## 使用指南

### 访问管理功能

1. 使用管理员账号登录系统
2. 登录后会自动显示"用户管理"标签
3. 点击"用户管理"查看所有用户

### 删除用户

1. 在用户列表中找到要删除的用户
2. 点击该用户行的"删除"按钮
3. 确认删除操作
4. 系统会自动刷新用户列表

注意：
- 管理员账号的删除按钮是禁用状态
- 删除操作不可恢复，请谨慎操作

## API 接口

### 获取用户列表
```http
GET /api/admin/users?is_admin=true
```

响应示例：
```json
[
  {
    "id": 1,
    "username": "admin",
    "email": null,
    "is_admin": true,
    "detection_count": 0,
    "created_at": "2026-03-20T10:00:00"
  },
  {
    "id": 2,
    "username": "user123",
    "email": "user@example.com",
    "is_admin": false,
    "detection_count": 0,
    "created_at": "2026-03-21T15:30:00"
  }
]
```

### 删除用户
```http
DELETE /api/admin/users/{user_id}?is_admin=true
```

响应示例：
```json
{
  "success": true,
  "message": "用户 user123 已删除"
}
```

### 获取系统统计
```http
GET /api/admin/stats?is_admin=true
```

响应示例：
```json
{
  "total_users": 10,
  "total_detections": 150
}
```

## 权限验证

所有管理员接口都需要验证管理员权限：
- 请求参数必须包含 `is_admin=true`
- 如果不是管理员，返回 403 错误

## 安全建议

1. **管理员账号安全**
   - 使用强密码
   - 定期更换密码
   - 不要共享管理员账号

2. **操作审慎**
   - 删除用户前确认身份
   - 定期备份数据库
   - 记录重要操作日志

3. **权限控制**
   - 限制管理员账号数量
   - 定期审查用户列表
   - 及时删除不活跃账号

## 常见问题

### Q: 如何创建管理员账号？
A: 注册时用户名包含 "admin" 即可自动识别为管理员。

### Q: 为什么不能删除管理员账号？
A: 为了防止误删导致系统无法管理，管理员账号不能通过界面删除。如需删除，请直接操作数据库。

### Q: 删除用户后数据会怎样？
A: 当前版本删除用户不会影响检测记录（因为检测记录表没有 user_id 字段）。

### Q: 如何查看用户的检测历史？
A: 当前版本暂不支持按用户查看检测历史。未来版本会添加此功能。

## 技术实现

### 后端实现
- 文件：`backend/app/routes/admin.py`
- 框架：FastAPI
- 权限验证：通过查询参数 `is_admin` 验证

### 前端实现
- 文件：`frontend/src/components/UserManagement.vue`
- 框架：Vue 3 + Element Plus
- 条件渲染：根据 `isAdmin` prop 显示/隐藏

### 数据库
- 表：`users`
- 字段：`id`, `username`, `password`, `email`, `security_question`, `security_answer`, `created_at`
- 管理员判断：`username LIKE '%admin%'`
