# services 目录说明

## 用途
存放业务逻辑服务层代码

## 文件说明

### yolo_service.py
YOLO 检测服务，提供核心检测功能：

#### 主要功能
1. **图片检测** (`predict_on_image`)
   - 使用 YOLO 模型检测商品
   - 自动推算空货架位置
   - 生成标注图片
   - 返回检测结果

2. **空货架检测算法**
   - 基于商品位置推算空位
   - 支持多层货架检测
   - 可配置间隙阈值和行聚类参数

#### 检测流程
```
上传图片 → YOLO检测商品 → 分析货架布局 → 推算空位 → 绘制标注 → 返回结果
```

#### 参数说明
- `conf_threshold`: 置信度阈值（默认 0.25）
- `iou_threshold`: IOU 阈值（默认 0.45）
- `gap_threshold`: 空位检测阈值（默认 0.5，即商品宽度的 50%）
- `row_threshold`: 行聚类阈值（默认 0.6，即商品高度的 60%）

## 服务层设计原则
1. **单一职责**：每个服务类负责一个特定功能
2. **可测试性**：业务逻辑独立，便于单元测试
3. **可复用性**：服务可被多个路由调用
4. **无状态**：服务方法应该是无状态的

## 添加新服务
1. 创建新的服务文件（如 `report_service.py`）
2. 定义服务类：
   ```python
   class ReportService:
       @staticmethod
       def generate_report(data):
           # 业务逻辑
           return result
   ```
3. 在路由中调用服务

## 注意事项
- 服务层不应直接处理 HTTP 请求/响应
- 复杂的业务逻辑应放在服务层，而非路由层
- 使用静态方法或类方法，避免不必要的实例化
- 添加适当的错误处理和日志记录
