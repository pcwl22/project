# 数据库更新说明

## 更新内容

为 `users` 表添加以下字段以支持忘记密码功能：

- `email` VARCHAR(255) - 邮箱（预留，可选）
- `security_question` VARCHAR(255) - 安全问题（必填）
- `security_answer` VARCHAR(255) - 安全答案（必填）

## 快速更新

### 方法一：使用自动更新脚本（推荐）

```bash
cd E:\bysj\ultralytics-8.2.27\project
conda activate yolo
python update_database.py
```

脚本会自动：
1. 连接数据库
2. 检查表结构
3. 添加缺失的字段
4. 显示更新结果
5. 检查现有用户状态
6. 可选：为现有用户设置默认安全问题

### 方法二：手动执行 SQL

```sql
-- 连接到数据库
USE ultralytics_auth;

-- 添加字段
ALTER TABLE users ADD COLUMN email VARCHAR(255) NULL;
ALTER TABLE users ADD COLUMN security_question VARCHAR(255) NULL;
ALTER TABLE users ADD COLUMN security_answer VARCHAR(255) NULL;

-- 验证字段已添加
DESCRIBE users;
```

## 现有用户处理

### 情况1：用户未设置安全问题

**问题**：现有用户无法使用忘记密码功能

**解决方案**：

#### 选项A：批量设置默认问题
```sql
UPDATE users 
SET security_question = '您的学号是？',
    security_answer = username
WHERE security_question IS NULL OR security_question = '';
```

#### 选项B：使用管理工具逐个设置
```bash
python manage_database.py
# 选择 3. 为用户设置安全问题
```

#### 选项C：通知用户重新注册
- 删除旧账户
- 用户重新注册并设置安全问题

### 情况2：管理员手动重置密码

如果用户忘记密码且未设置安全问题：

```bash
python manage_database.py
# 选择 4. 重置用户密码
```

或直接 SQL：
```sql
UPDATE users 
SET password = '新密码'
WHERE username = '用户名';
```

```sql

    CASE

    END as security_status
FROM users;
```

## 常见问题

### Q1: 更新脚本报错 "Access denied"
**A**: 检查数据库密码是否正确，修改脚本中的 `password` 配置

### Q2: 字段已存在错误
**A**: 字段已经添加过了，可以忽略或删除后重新添加

### Q3: 现有用户无法登录
**A**: 更新不会影响现有用户登录，只是他们无法使用忘记密码功能

### Q4: 如何批量导出用户数据
**A**: 
```sql
SELECT * FROM users INTO OUTFILE '/tmp/users_backup.csv'
FIELDS TERMINATED BY ',' 
ENCLOSED BY '"'
LINES TERMINATED BY '\n';
```
