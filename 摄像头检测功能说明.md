# 摄像头实时检测功能说明

## 功能概述

摄像头实时检测功能允许用户使用电脑摄像头或外接摄像头进行实时货架检测，适用于现场监控和实时分析场景。

## 核心特性

✅ **实时检测**：支持实时视频流检测  
✅ **多摄像头支持**：自动识别并支持切换多个摄像头  
✅ **参数可调**：实时调整置信度、IOU 阈值和检测间隔  
✅ **实时统计**：显示商品数、空货架数、缺货率和 FPS  
✅ **快照保存**：一键保存当前检测画面  
✅ **低延迟**：优化的检测流程，确保流畅体验  

## 使用指南

### 1. 启动摄像头检测

1. 登录系统
2. 点击左侧菜单"摄像头检测"
3. 系统自动启动默认摄像头
4. 允许浏览器访问摄像头权限

### 2. 选择摄像头

如果有多个摄像头设备：
1. 在"摄像头设备"下拉框中选择
2. 系统自动切换到选中的摄像头

### 3. 调整检测参数

**置信度阈值**（默认 0.25）
- 范围：0.1 - 1.0
- 说明：检测结果的最低置信度
- 建议：标准场景 0.25，高精度 0.4

**IOU 阈值**（默认 0.45）
- 范围：0.1 - 1.0
- 说明：重叠框的合并阈值
- 建议：标准场景 0.45，密集场景 0.3

**检测间隔**（默认 1.0 秒）
- 范围：0.5 - 5.0 秒
- 说明：两次检测之间的时间间隔
- 建议：实时性要求高 0.5s，性能优先 2.0s

### 4. 开始检测

1. 调整好参数后，点击"开始检测"按钮
2. 系统开始实时检测并在视频上绘制检测框
3. 绿色框：商品
4. 红色框：空货架
5. 右上角显示检测状态

### 5. 查看统计信息

检测过程中，底部显示实时统计：
- **商品数**：当前检测到的商品数量
- **空货架**：当前检测到的空货架数量
- **缺货率**：空货架占比（超过 20% 显示红色警告）
- **FPS**：每秒处理帧数

### 6. 拍摄快照

1. 点击"拍摄快照"按钮
2. 系统保存当前检测画面到本地
3. 文件名格式：`snapshot_时间戳.jpg`

### 7. 停止检测

点击"停止检测"按钮即可停止实时检测。

## 技术实现

### 前端技术
- **WebRTC API**：访问摄像头设备
- **Canvas API**：绘制检测结果
- **MediaDevices API**：枚举和管理摄像头

### 检测流程
```
摄像头 → 视频流 → 捕获帧 → 发送后端 → YOLO检测 → 返回结果 → 绘制画面
```

### 性能优化
1. **可调检测间隔**：避免过度请求
2. **异步处理**：不阻塞视频流
3. **画布复用**：减少内存开销
4. **JPEG 压缩**：减少传输数据量

## 常见问题

### Q1: 无法访问摄像头
**A**: 检查以下几点：
1. 浏览器是否有摄像头权限
2. 摄像头是否被其他程序占用
3. 使用 HTTPS 或 localhost 访问

### Q2: 检测延迟太高
**A**: 尝试以下方法：
1. 增加检测间隔（如 2 秒）
2. 降低摄像头分辨率
3. 关闭其他占用资源的程序

### Q3: 检测不准确
**A**: 调整参数：
1. 提高置信度阈值（如 0.4）
2. 调整 IOU 阈值
3. 确保光线充足

### Q4: 画面卡顿
**A**: 优化设置：
1. 增加检测间隔
2. 降低视频分辨率
3. 检查网络连接

### Q5: 快照保存失败
**A**: 检查：
1. 浏览器下载权限
2. 磁盘空间是否充足
3. 是否有文件写入权限

## 最佳实践

### 1. 摄像头位置
- 正对货架，避免倾斜
- 距离适中，确保清晰度
- 光线充足，避免逆光

### 2. 参数设置
- 初次使用：默认参数
- 高精度场景：置信度 0.4，间隔 2s
- 实时性场景：置信度 0.25，间隔 0.5s

### 3. 网络环境
- 使用有线网络（推荐）
- 确保网络稳定
- 避免高峰时段

### 4. 设备选择
- 使用高清摄像头
- 确保设备性能充足
- 定期清理摄像头镜头

## 故障排查

### 问题：摄像头列表为空
**解决方案**：
1. 刷新页面
2. 检查设备管理器
3. 重新插拔摄像头
4. 重启浏览器

### 问题：检测结果不显示
**解决方案**：
1. 检查后端服务是否运行
2. 查看浏览器控制台错误
3. 确认网络连接正常
4. 重启检测

### 问题：FPS 过低
**解决方案**：
1. 增加检测间隔
2. 降低图片质量
3. 关闭其他标签页
4. 升级硬件配置
