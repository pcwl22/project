# 货架商品检测与空货架识别系统

## 项目概述

基于 YOLOv8 的智能货架检测系统，能够自动识别货架上的商品并智能推算空货架位置，帮助零售商实时监控货架缺货情况，提高补货效率。

## 核心功能

### 1. 智能检测
- **商品检测**：使用 YOLOv8 深度学习模型识别货架上的商品
- **空货架推算**：基于商品位置智能推算空货架区域
- **多层货架支持**：自动识别多层货架结构
- **实时标注**：在图片上绘制检测框和标签

### 2. 用户管理
- **用户注册/登录**：支持多用户系统，注册时必须设置安全问题
- **忘记密码**：通过安全问题验证重置密码
- **权限管理**：区分普通用户和管理员
- **会话管理**：安全的用户会话控制

### 3. 历史记录
- **自动保存**：每次检测自动保存到数据库
- **记录查询**：支持分页浏览历史记录
- **详情查看**：查看检测详情和统计信息
- **图片管理**：下载原图和标注图
- **批量操作**：支持批量删除记录

### 4. 模型管理
- **模型上传**：支持上传自定义 YOLOv8 模型
- **模型切换**：灵活切换不同的检测模型
- **参数调整**：实时调整置信度和 IOU 阈值

### 5. 统计分析
- **缺货率计算**：自动计算货架缺货率
- **商品统计**：统计商品数量和分布
- **可视化展示**：直观的统计图表

### 6. 视频检测
- **视频处理**：支持监控视频逐帧检测
- **参数配置**：可调整置信度和 IOU 阈值
- **结果导出**：生成带标注的检测结果视频
- **格式支持**：支持 MP4、AVI、MOV 等常见格式

### 7. 摄像头检测
- **实时检测**：支持摄像头实时检测
- **设备切换**：自动检测并切换可用摄像头
- **参数调整**：实时调整检测间隔和阈值
- **快照保存**：捕获并保存检测快照
- **性能监控**：显示 FPS 和检测统计

## 核心算法

### 空货架检测算法

系统采用智能推算算法，基于已检测到的商品位置推算空货架区域：

1. **商品检测**：使用 YOLOv8 检测所有商品
2. **行聚类**：根据 Y 坐标将商品分组到不同货架层
3. **间隙分析**：分析每层商品之间的水平间隙
4. **空位推算**：当间隙超过阈值时，推算为空货架
5. **结果标注**：在图片上标注空货架区域

#### 算法参数
- `conf_threshold`：置信度阈值（默认 0.25）
- `iou_threshold`：IOU 阈值（默认 0.45）
- `gap_threshold`：空位检测阈值（默认 0.5，即商品宽度的 50%）
- `row_threshold`：行聚类阈值（默认 0.6，即商品高度的 60%）

#### 算法优势
- 无需额外训练空货架样本
- 适应不同货架布局
- 准确率高，误报率低
- 实时处理速度快

## 数据库设计

### users 表（用户信息）
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(150) UNIQUE NOT NULL,
    password TEXT NOT NULL,
    email VARCHAR(255),                      -- 邮箱（可选）
    security_question VARCHAR(255),          -- 安全问题（必填）
    security_answer VARCHAR(255),            -- 安全答案（必填）
    is_admin TINYINT(1) DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### detection_results 表（检测记录）
```sql
CREATE TABLE detection_results (
    id INT PRIMARY KEY AUTO_INCREMENT,
    image_path VARCHAR(512) NOT NULL,
    saved_image_path VARCHAR(512),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    user_id INT
);
```

### detection_boxes 表（检测框详情）
```sql
CREATE TABLE detection_boxes (
    id INT PRIMARY KEY AUTO_INCREMENT,
    result_id INT NOT NULL,
    cls_name VARCHAR(64) NOT NULL,
    x1 FLOAT NOT NULL,
    y1 FLOAT NOT NULL,
    x2 FLOAT NOT NULL,
    y2 FLOAT NOT NULL,
    score FLOAT NOT NULL,
    extra TEXT,
    FOREIGN KEY (result_id) REFERENCES detection_results(id) ON DELETE CASCADE
);
```

## API 接口

### 认证接口
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/register` - 用户注册（必须提供安全问题和答案）
- `POST /api/auth/forgot-password` - 忘记密码（验证用户名，返回安全问题）
- `POST /api/auth/reset-password` - 重置密码（验证安全答案，设置新密码）

### 检测接口
- `POST /upload_model` - 上传模型
- `POST /detect` - 图片检测

### 历史记录接口
- `GET /api/detections/` - 获取历史列表
- `GET /api/detections/{id}` - 获取记录详情
- `DELETE /api/detections/{id}` - 删除记录
- `GET /api/detections/{id}/original` - 获取原图
- `GET /api/detections/{id}/annotated` - 获取标注图
