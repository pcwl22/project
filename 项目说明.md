# 项目说明

## 1. 项目名称

ShelfDetect：基于 YOLOv8 的货架商品检测与空货架识别系统

## 2. 项目定位

本项目是一个面向零售货架巡检场景的智能检测系统，核心目标是利用计算机视觉技术自动识别货架上的商品和空货位情况，并通过前后端分离的方式提供可操作、可展示、可追踪的业务系统。

与只做模型训练或单张图片推理的实验项目不同，这个项目已经实现了从检测模型、后端接口、数据库存储到前端交互页面的完整闭环，适合作为课程设计、毕业设计和项目演示系统使用。

## 3. 项目背景与意义

传统超市、便利店和仓储货架巡检主要依赖人工完成，存在以下问题：

- 人工巡检效率低，无法长时间高频监测
- 不同人员的判断标准不一致，结果主观性较强
- 发现缺货或陈列异常往往不够及时
- 检测结果难以系统化留存和统计分析

本项目通过 YOLOv8 目标检测结合空货架推断算法，实现对货架状态的自动化检测，能够提升盘点效率、辅助补货决策，并为后续的数据分析与业务扩展提供基础。

## 7. 系统架构说明

本项目采用前后端分离架构：

1. 前端使用 Vue 3 构建用户界面，负责登录、文件上传、结果展示、参数配置与历史记录交互。
2. 后端使用 FastAPI 提供 RESTful API，负责接收图片/视频、调用 YOLOv8 推理、保存结果、读写数据库。
3. 数据库用于保存用户、检测记录和检测框信息。
4. 本地文件系统用于保存原始图片、结果图片、视频和摄像头检测输出。

整体调用流程为：

用户操作页面 -> 前端发起请求 -> FastAPI 接口接收数据 -> YoloService 执行检测 -> 写入数据库/文件 -> 返回结果给前端展示

## 9. 空货架检测算法说明

该项目最有特色的部分不只是目标检测，而是 `backend/app/services/yolo_service.py` 中的空货架推断逻辑。

从代码实现来看，空货架检测流程大致如下：

1. 使用 YOLOv8 检测图片中的商品与可能存在的空货架类别
2. 对检测结果进行黑边区域过滤
3. 将商品按垂直位置聚类为多行
4. 在同一行内按照横向间距寻找异常空隙
5. 对两侧边缘区域进行空位检测
6. 去除与商品重叠的误判空货架框
7. 统计商品数量、空货架数量、总货位数与空置率

算法中可调参数包括：

- `gap_threshold`
- `row_threshold`
- `min_gap_pixels`
- `edge_detection`
- `min_products_per_row`
- `black_edge_threshold`

这些参数会通过 `ConfigManager` 持久化到 `algorithm_config.json`。

## 13. 当前实现中的特点与限制

基于实际代码，这个项目有一些值得在说明里明确写出的实现特点：

- 使用用户名中包含 `admin` 的方式判断管理员身份，实现简单但不够严格
- 登录密码兼容明文和 MD5 校验，说明系统处于开发/演示阶段
- 部分接口虽然接收 `user_id` 和 `is_admin` 参数，但历史查询过滤逻辑仍较简化
- 配置文件与部分资源路径仍为本机绝对路径，换机器部署前需要调整
- 前端接口地址存在相对路径和 `http://localhost:8000` 混用情况
- 根目录仍保留一个旧版 `main.py`，实际运行建议使用 `backend/app/main.py`

## 14. 项目亮点总结

从完整项目角度看，当前系统的亮点主要体现在：

- 不仅有目标检测模型，还有与货架业务结合的空货架推断逻辑
- 支持图片、视频、摄像头三种输入方式，场景覆盖较完整
- 前后端分离，具备用户、模型、历史、参数等完整功能模块
- 检测结果可以存储和回查，不只是一次性推理页面
