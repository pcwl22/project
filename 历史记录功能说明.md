# 检测历史记录功能说明

## 功能概述

系统现在会自动将所有检测结果保存到 MySQL 数据库中，用户可以在"历史记录"页面查看和管理。

## 数据库表结构

### 1. detection_results (检测结果表)
- `id`: 主键
- `image_path`: 原始图片路径
- `saved_image_path`: 标注后图片路径
- `created_at`: 创建时间

### 2. detection_boxes (检测框表)
- `id`: 主键
- `result_id`: 关联的检测结果ID
- `cls_name`: 类别名称 (product/empty_shelf)
- `x1, y1, x2, y2`: 检测框坐标
- `score`: 置信度
- `extra`: 额外信息

## API 接口

### 1. 图片检测 (自动保存)
```
POST /detect?model=best.pt&conf=0.25&iou=0.45&user_id=14
```
- 每次检测都会自动保存到数据库
- user_id 参数会自动从登录信息获取

### 2. 获取历史记录列表
```
GET /detections/?skip=0&limit=20&user_id=14&is_admin=false
```
返回格式：
```json
{
  "success": true,
  "total": 100,
  "records": [
    {
      "id": 1,
      "image_path": "...",
      "saved_image_path": "...",
      "created_at": "2026-03-18T12:00:00",
      "total_detections": 110,
      "product_count": 100,
      "empty_count": 10,
      "empty_rate": 0.091
    }
  ]
}
```

### 3. 获取记录详情
```
GET /detections/{record_id}?user_id=14&is_admin=false
```

### 4. 删除记录
```
DELETE /detections/{record_id}?user_id=14&is_admin=false
```

### 5. 获取图片
```
GET /detections/{record_id}/original  # 原图
GET /detections/{record_id}/annotated # 标注图
```

## 前端使用

### 1. 图片检测
- 在"图片检测"页面上传图片
- 系统自动检测并保存结果
- 检测完成后可在"历史记录"查看

### 2. 查看历史
- 点击"历史记录"标签页
- 查看所有检测记录
- 支持分页浏览

### 3. 查看详情
- 点击记录可查看详细信息
- 查看原图和标注图
- 查看所有检测框信息

### 4. 删除记录
- 单条删除：点击删除按钮
- 批量删除：选择多条记录后批量删除
- 删除会同时删除数据库记录和图片文件

## 注意事项

1. **自动保存**：所有检测都会自动保存，无需手动操作
2. **数据持久化**：数据保存在 MySQL 数据库中，重启服务不会丢失
3. **图片存储**：原图和标注图都保存在 `test/api` 目录
4. **级联删除**：删除检测记录时，关联的检测框会自动删除
5. **文件清理**：删除记录时会同时删除对应的图片文件

## 故障排查

### 问题1：历史记录为空
- 检查是否进行过检测
- 检查 MySQL 服务是否运行
- 查看后端日志是否有错误

### 问题2：无法保存记录
- 检查 MySQL 连接配置
- 确认数据库表已创建
- 查看后端日志错误信息

### 问题3：图片无法显示
- 检查图片文件是否存在
- 确认文件路径正确
- 检查文件权限

## 数据库维护

### 查看记录数量
```sql
SELECT COUNT(*) FROM detection_results;
SELECT COUNT(*) FROM detection_boxes;
```

### 清空历史记录
```sql
DELETE FROM detection_boxes;
DELETE FROM detection_results;
```

### 查看最近记录
```sql
SELECT * FROM detection_results 
ORDER BY created_at DESC 
LIMIT 10;
```
